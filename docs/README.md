# توثيق مشروع ERP

## وصف عام
نظام الإدارة المتكامل (ERP) هو حل شامل لإدارة جميع جوانب العمل بكفاءة وسهولة. يتكون المشروع من جزئين رئيسيين:
- **الخلفية (Back-end):** مبني باستخدام PHP وإطار Slim، ويحتوي على منطق الأعمال وواجهات برمجة التطبيقات (APIs).
- **الواجهة الأمامية (Front-end):** مبنية باستخدام Vue.js وتستخدم Vite وTailwind CSS.

---

## مكونات المشروع

### الخلفية (Back-end)
- **api/v1/index.php**: نقطة الدخول الرئيسية للـ API.
- **api/v1/routes.php**: تعريف جميع المسارات (Routes) الخاصة بالـ API.
- **api/v1/handlers/**: يحتوي على جميع الـ Handlers لمعالجة الطلبات المختلفة (مثل المستخدمين، المنتجات، التقارير، ...).
- **api/v1/src/**: الخدمات (Services) والكلاسات المساعدة.
- **config/**: إعدادات الاتصال بقاعدة البيانات، التوثيق، الأمان، وغيرها.

### الواجهة الأمامية (Front-end)
- **erp-frontend/src/**: جميع ملفات المصدر الخاصة بالواجهة.
  - **components/**: مكونات Vue القابلة لإعادة الاستخدام.
  - **views/**: الصفحات الرئيسية للتطبيق.
  - **router/**: تعريف المسارات والتنقل بين الصفحات.
  - **store/** و **stores/**: إدارة الحالة باستخدام Pinia.
  - **services/**: التعامل مع الـ APIs.

## كيفية التشغيل

### الخلفية
1. تأكد من وجود PHP 8.0 أو أعلى.
2. ثبت الاعتماديات عبر Composer:
   ```bash
   composer install
   ```
3. قم بضبط إعدادات قاعدة البيانات في `config/database.php`.
4. شغل السيرفر المحلي (مثلاً XAMPP أو Apache).

### الواجهة الأمامية
1. تأكد من وجود Node.js.
2. ثبت الاعتماديات:
   ```bash
   npm install
   ```
3. شغل التطبيق:
   ```bash
   npm run dev
   ```

## اقتراحات لتعليق الكود داخليًا
- استخدم تعليقات توضيحية أعلى كل دالة أو كلاس لشرح وظيفتها.
- وضّح المتغيرات المهمة وأي منطق معقد داخل الكود.
- يمكنك استخدام أدوات مثل phpDocumentor (للـ PHP) وJSDoc (للـ JavaScript/Vue) لتوليد توثيق تلقائي.

---

## دليل التعليق الداخلي للملفات

فيما يلي قائمة بأهم الملفات والمجلدات التي يجب إضافة تعليقات توثيقية داخلها، مع نوع التعليق المقترح:

### الخلفية (Back-end)
- **api/v1/index.php**
  - تعليق في أعلى الملف يوضح أنه نقطة الدخول للـ API.
  - تعليقات توضح منطق التهيئة وأي إعدادات رئيسية.
- **api/v1/routes.php**
  - تعليق يوضح وظيفة كل Route.
  - توثيق كل مجموعة مسارات (Users, Products, ...).
- **api/v1/handlers/**
  - في كل ملف Handler: تعليق أعلى كل دالة يشرح وظيفتها.
  - توثيق المتغيرات المهمة والمنطق المعقد.
- **api/v1/src/**
  - توثيق كل كلاس أو خدمة (Service) والغرض منها.
  - شرح مختصر لكل دالة عامة.
- **config/**
  - توثيق إعدادات كل ملف (قاعدة البيانات، الأمان، ...)، وشرح الخيارات المتاحة.

### الواجهة الأمامية (Front-end)
- **erp-frontend/src/components/**
  - في كل مكون Vue: تعليق أعلى الملف يوضح وظيفة المكون.
  - توثيق Props وMethods المهمة.
- **erp-frontend/src/views/**
  - تعليق يوضح الغرض من كل View.
- **erp-frontend/src/router/**
  - توثيق المسارات الرئيسية وأي Middleware مستخدم.
- **erp-frontend/src/services/**
  - توثيق كل خدمة تتعامل مع الـ API.
  - شرح الدوال العامة.
- **erp-frontend/src/store/** و **stores**
  - توثيق المتغيرات الرئيسية (state) والدوال (actions, getters).

## نماذج لتعليقات توثيقية (Documentation Comments)

### للملفات البرمجية بلغة PHP (phpDoc)
```php
/**
 * وصف مختصر لوظيفة الملف أو الكلاس.
 *
 * تفاصيل إضافية عند الحاجة.
 */

/**
 * وصف الدالة
 *
 * @param string $name اسم المستخدم
 * @param int $age عمر المستخدم
 * @return bool
 */
function exampleFunction($name, $age) {
    // ...
}
```

### لمكونات Vue أو ملفات JavaScript (JSDoc)
```js
/**
 * وصف مختصر لوظيفة المكون أو الدالة.
 *
 * @param {string} name اسم المستخدم
 * @param {number} age عمر المستخدم
 * @returns {boolean}
 */
function exampleFunction(name, age) {
  // ...
}
```

```js
/**
 * مكون Vue لعرض بيانات المستخدم
 * @props {String} name - اسم المستخدم
 */
export default {
  props: ['name'],
  // ...
}
```

استخدم هذه النماذج في جميع الملفات المذكورة في دليل التعليق الداخلي، مع تخصيص الوصف والمعاملات حسب الحاجة.
