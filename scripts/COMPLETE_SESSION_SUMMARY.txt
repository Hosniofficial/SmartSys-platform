                              PHASE 6 - LEDGER BUG FIX
═══════════════════════════════════════════════════════════════════════════════════════════════════

PROBLEM:
────────
Return #339 (sales_return_only) was showing an EXTRA debit line "صرف نقدي للعميل" in ledger.
This line should only appear for refunds with transaction_subtype='sales_return_refund'.
Impact: Closing balance showed 2,000 instead of correct 0.00

ROOT CAUSE:
───────────
File: api/v1/handlers/AccountStatementHandler.php
Method: getCustomerReferences() - Payment handling section

The SQL query did NOT fetch return's refund_amount and refund_method,
so the code automatically assumed ALL refund payments are 'sales_return_refund'.

SOLUTION:
─────────
1. Added LEFT JOIN to returns table in SQL query
2. Selected r.refund_amount and r.refund_method
3. Updated transaction_subtype logic:
   - If refund_amount >= 0.01 AND refund_method='cash' → 'sales_return_refund'
   - If refund_amount >= 0.01 AND refund_method='bank_transfer' → 'sales_return_bank_refund'
   - If refund_amount < 0.01 → null (sales_return_only)

VERIFICATION:
─────────────
✓ Test data confirms: Returns #337, #338, #339 all have refund_amount=0, refund_method=NULL
✓ Expected result: transaction_subtype = null (sales_return_only)
✓ Expected ledger: ONLY credit line, NO debit line
✓ Expected balance: 0.00 ✓

                          PHASE 7 - refund_mode='auto' BUG FIX
═══════════════════════════════════════════════════════════════════════════════════════════════════

PROBLEM:
────────
When creating a return for a cash invoice with refund_mode='auto':
  ❌ The system did NOT deduct the return amount from other customer debts
  ❌ Customer debt was NOT reduced even though 'auto' should deduce first then refund excess

When using refund_mode='credit_note':
  ✓ Same operation works correctly (debt IS deducted)

ROOT CAUSE:
───────────
File: api/v1/src/Services/ReturnService.php
Line 758-762: When invoice is fully paid (saleOutstanding=0), the code had:

  if ($refundMode === 'credit_note' || $refundMode === 'deduct_and_return') {
      $data['paid_amount'] = 0;
  }

The 'auto' mode was NOT included! This caused:
  1. paid_amount was not set to 0
  2. Later logic at line 774+ might not execute properly
  3. deductFromCustomerBalance remained uninitialized
  4. allocateCustomerBalance was never called

SOLUTION:
─────────
Made 3 changes to ReturnService.php:

1. LINE 758: Added 'auto' to the condition
   NEW: if ($refundMode === 'credit_note' || $refundMode === 'deduct_and_return' || $refundMode === 'auto')
   
2. LINES 774-790: Improved deductFromCustomerBalance handling
   - For 'auto': deductFromCustomerBalance = min(customerTotalOutstanding, grandTotal)
   - For 'cash': paid_amount = grandTotal, deductFromCustomerBalance = 0
   - For 'credit_note': paid_amount = 0, deductFromCustomerBalance = 0
   - Default: same as 'credit_note'
   
3. LINES 825-834: Added 'auto' handling for purchase returns
   - For 'auto': paid_amount = 0 (consistent with sales returns)

HOW IT WORKS NOW:
─────────────────
Scenario: Cash invoice #784 (2000, paid) + Return + Customer has debt on #785 (3000)

1. Create return for #784 with refund_mode='auto'
   - saleOutstanding = 0 (fully paid)
   - Line 759: NOW handles 'auto' ✓
   - paid_amount = 0 ✓
   
2. Calculate deductFromCustomerBalance
   - Line 776: customerTotalOutstanding = 3000 (from other debts)
   - deductFromCustomerBalance = min(3000, 2000) = 2000
   - paid_amount = max(0, 2000 - 2000) = 0
   
3. Save return and call allocateCustomerBalance
   - Deduct 2000 from #785
   - #785 outstanding: 3000 - 2000 = 1000 ✓
