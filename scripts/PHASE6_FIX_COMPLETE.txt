✅ ISSUE FIXED: Return #339 Ledger Display Bug

PROBLEM:
────────
Return #339 (sales_return_only) was showing an EXTRA debit line "صرف نقدي للعميل" in the ledger.
This line should only appear for returns with actual cash refunds (sales_return_refund).
Result: Closing balance incorrectly showed 2,000 instead of 0.00

ROOT CAUSE:
───────────
File: api/v1/handlers/AccountStatementHandler.php
Method: getCustomerReferences() → Payment handling section

The code automatically set transaction_subtype='sales_return_refund' for ALL refund payments,
WITHOUT checking the return's actual refund_amount or refund_method.

═══════════════════════════════════════════════════════════════════════════════════════════════════
                                   SOLUTION APPLIED
═══════════════════════════════════════════════════════════════════════════════════════════════════

CHANGES MADE:
─────────────

1. SQL Enhancement:
   ├─ Added LEFT JOIN to returns table in payments query
   ├─ Selected r.refund_amount and r.refund_method fields
   └─ These values now flow through to payment array transformation

2. Logic Fix:
   OLD:  'transaction_subtype' => $paymentType === 'refund' ? 'sales_return_refund' : null
   
   NEW:  Conditional logic that checks:
         • IF paymentType = 'refund' AND refund_amount >= 0.01
           ├─ IF refund_method = 'cash' → 'sales_return_refund'
           ├─ ELSE IF refund_method = 'bank_transfer' → 'sales_return_bank_refund'
         • ELSE → null (sales_return_only)
